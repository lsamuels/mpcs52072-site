    Assignment Focus¶

    This assignment is designed to help you become familiar with common GPU programming patterns, including:

    • Prefix scan

    • Histogram computation

    • Warp-level primitives

    The primary goal is to build a correct and functional implementation of these patterns in a realistic pipeline.

    You are not required to profile or optimize performance for this assignment.

    While you will implement multiple GPU techniques (e.g., shared memory and warp-level primitives), the emphasis is on understanding how these patterns work and not on achieving maximum performance.

    We will return to performance analysis and profiling in future assignments.

    Warp-Level Primitives¶

    In this assignment, you will use warp-level primitives to improve the efficiency of your GPU implementation.

    Two important primitives are:

    • __ballot_sync

    • __popc

    __ballot_sync¶

    unsigned mask = __ballot_sync(0xffffffff, predicate);

    This function evaluates predicate for all threads in the warp and returns a 32-bit integer (mask).

    • Each bit corresponds to a thread (lane) in the warp

    • If a thread’s predicate is true, its bit is set to 1

    • Otherwise, the bit is 0

    Example:

    lane:        0  1  2  3  4  5  6  7
predicate:   1  0  1  1  0  0  1  0

    The resulting mask encodes which threads satisfied the condition.

    __popc¶

    int count = __popc(mask);

    This function counts the number of bits set to 1 in mask.

    Using the previous example:

    count = 4

    This means 4 threads in the warp satisfied the condition.

    Computing a Local Rank¶

    You can compute how many threads before the current thread satisfy the condition:

    int lane = threadIdx.x % warpSize;
int local_rank = __popc(mask & ((1u << lane) - 1));

    This gives each thread a position among the threads where predicate == true.

    Example Kernel¶

    __global__ void warp_example(int* data, int* output, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        int is_even = (data[idx] % 2 == 0);

        unsigned mask = __ballot_sync(0xffffffff, is_even);
        int lane = threadIdx.x % warpSize;

        int total_in_warp = __popc(mask);
        int local_rank = __popc(mask & ((1u << lane) - 1));

        output[idx * 2]     = total_in_warp;
        output[idx * 2 + 1] = local_rank;
    }
}

    For each thread:

    • total_in_warp = number of threads in the warp where the condition is true

    • local_rank = number of threads before this one that satisfy the condition

    Assignment Overview¶

    In this assignment, you will implement a GPU-based pipeline for processing a large stream of social media posts. The goal is to identify “trending” posts and compute summary statistics using histograms. The following subsections explain the structure of the data, formulas for computing various statistics and the overall pipeline structure for implementing the program.

    Posts Format¶

    Your program will read social media post input from a file with the following format:

    Num_Of_Posts
post_id
user_id
content_type
likes
shares
comments
...

    • The first line specifies the total number of posts.

    • Each post is described by 6 consecutive lines.

    • There are exactly Num_Of_Posts posts in the file.

    • The file will be provided as an argument to the program.

    The program will construct Post structs. (hw5/include/post.h) based off reading in the data from the file.

    struct Post {
    int post_id;
    int user_id;
    int content_type;   // 0=text, 1=image, 2=video, 3=link
    int likes;
    int shares;
    int comments;
};

    Trending Definition¶

    Each post is assigned an engagement score:

    score = likes + 2 * shares + 3 * comments;

    A post is considered trending if:

    score >= threshold;

    The threshold value will be provided as an argument to the program.

    Pipeline¶

    The structure for the application will be as follows:

    Input File
   ↓
Load Posts (CPU)
   ↓
Copy to GPU
   ↓
Compute Score + Flag
   ↓
Prefix Scan
   ↓
Compute Histograms (Shared Memory/Warp Prims)
   ↓
Copy Results Back
   ↓
Print Output

    Command Line Interface¶

    The program’s usage is as follows:

    ./feed_filter <input_file> <threshold> <block_size>

    Arguments:

    • input_file Path to the input file containing post data.

    • threshold Integer value used to determine whether a post is trending. A post is considered trending if:

      score >= threshold

    • block_size Number of threads per block used when launching CUDA kernels.

    You can assume we will provide valid and correctly ordered arguments. You do not need to error test the command line arguments.

    Output¶

    Each line of output represents a summary of the processed posts.

    Line 1: Trending Posts Count

    • The total number of posts whose engagement score meets or exceeds the threshold.

    —

    Line 2: Content Type Histogram

    TEXT_COUNT,IMAGE_COUNT,VIDEO_COUNT,LINK_COUNT

    • TEXT_COUNT: number of posts with content_type = 0

    • IMAGE_COUNT: number of posts with content_type = 1

    • VIDEO_COUNT: number of posts with content_type = 2

    • LINK_COUNT: number of posts with content_type = 3

    This histogram counts all posts, not just trending ones.

    —

    Line 3: Engagement Histogram

    ENG_0_99_COUNT,ENG_100_499_COUNT,ENG_500_999_COUNT,ENG_1000_PLUS_COUNT

    • ENG_0_99_COUNT: number of posts with engagement score between 0 and 99

    • ENG_100_499_COUNT: number of posts with engagement score between 100 and 499

    • ENG_500_999_COUNT: number of posts with engagement score between 500 and 999

    • ENG_1000_PLUS_COUNT: number of posts with engagement score 1000 or greater

    This histogram also counts all posts, not just trending ones.

    Task 2: Host-Side Setup¶

    You will need device arrays for:

    • Posts

    • Flags

    • Scan results

    • Content histogram (size 4)

    • Engagement histogram (size 4)

    Task 3: Flagging Kernel (GPU)¶

    Compute a binary flag array:

    flags[i] = 1 if post i is trending
flags[i] = 0 otherwise

    Each thread should:

    • Compute the engagement score

    • Compare against the threshold

    • Write 0 or 1 into the flags array

    Task 4: Prefix Scan (GPU)¶

    Implement an exclusive prefix scan over the flags array.

    Definition:

    scan[i] = number of 1s in flags[0 ... i-1]

    Example:

    flags: [0, 1, 1, 0, 1]
scan:  [0, 0, 1, 2, 2]

    Requirements:

    • Must be an exclusive scan

    • You may implement this using one or more kernels

    • You may not use Thrust or other libraries

    Note on Prefix Scan Usage¶

    the prefix scan is used to compute the total number of trending posts.

    Given a binary flag array:

    flags[i] = 1 if post i is trending flags[i] = 0 otherwise

    The exclusive prefix scan produces:

    scan[i] = number of trending posts before index i

    The total number of trending posts can then be computed as:

    total = scan[n-1] + flags[n-1];

    This allows you to compute the total number of selected elements without using atomic operations.

    Although prefix scan is often used for stream compaction (filtering), you will explore that use case in a future assignment.

    Task 5: Histogram Kernels¶

    You must implement two versions of the histogram kernel:

    1. Shared Memory Histogram (Baseline)

      • Uses shared memory privatization

      • Uses atomic operations in shared and global memory

    2. Warp-Optimized Histogram

      • Uses __ballot_sync and __popc

      • Reduces redundant atomic operations by aggregating work within a warp

    The warp-optimized version should demonstrate that threads within a warp can cooperate before updating histogram bins.
