    Grayscale Conversion and Histogram (CUDA) Overview¶

    In this assignment, you will implement a CUDA-based program that processes images in two stages:

    1. Convert a color image (PNG) to grayscale

    2. Compute a histogram of grayscale pixel intensities and print the result

    You will work with real image data and use Unix tools to validate and visualize your results.

    Directory Description¶

    • bin/ Contains the compiled executable (grayscale).

    • data/ Contains input images you can use for testing. You will convert these color images into grayscale and write the results to: data/output/

    • include/png_flatten.h and src/png_flatten.c Provide helper functions for loading and saving PNG images.

    • src/grayscale.cu Main CUDA source file where you will implement your solution.

    • Makefile Used to build the project.

    • script/hw2-job.sh Example script for running your program on the cluster.

    PNG Handling (Provided Code)¶

    The repository includes helper functions for working with PNG images.

    1. png_flatten_load (in png_flatten.h) Loads a PNG image and returns:

      • A flattened 1D array of pixel data

      • Image width

      • Image height

    2. png_flatten_save (in png_flatten.h) Saves a flattened image array back to a PNG file.

    You should treat these as black-box utilities.

    Warning

    Do not modify png_flatten.h or png_flatten.c. These files are provided to simplify image I/O and must remain unchanged.

    Image Representation¶

    Images are stored as a flattened 1D array:

    unsigned char* image;

    Each pixel consists of 4 components:

    • Red (R)

    • Green (G)

    • Blue (B)

    • Alpha (A)

    These are stored consecutively in memory:

    [R, G, B, A,  R, G, B, A,  R, G, B, A, ...]

    Therefore: - Each pixel occupies 4 bytes - The starting index of pixel i is:

    int index = i * 4;

    You can see how this indexing works by examining the provided image_to_grayscale function.

    Grayscale Conversion¶

    The file src/grayscale.cu includes a host-side function:

    image_to_grayscale(...)

    This function converts a flattened RGB image into grayscale.

    The program: ma 1. Loads an input PNG file 2. Converts it to grayscale 3. Saves the result to the specified output file

    The executable takes two arguments:

    ./bin/grayscale <input.png> <output.png>

    For example:

    ./bin/grayscale data/test_img.png data/output/test_img_out.png

    For the remainder of this assignment, you will augment grayscale.cu to perform additional processing.

    Important Notes¶

    • You must operate on the flattened image representation

    • You must not modify the PNG helper files

    • All new functionality should be implemented inside grayscale.cu

    Grayscale Program: Usage and Behavior¶

    Your grayscale.cu program must support the following command-line usage:

    ./grayscale [-g BLOCK] input.png output.png

    The purpose of this program is to convert a color image (input.png) into a grayscale image (output.png), using either a sequential CPU implementation or a parallel GPU implementation.

    The command-line arguments modify how the program runs:

    • Default (no flags):

      Runs the sequential (CPU) version of the grayscale algorithm. This version reads input.png, converts it to grayscale using host code, and saves the result to output.png.

      Note

      Do not produce the histogram in the CPU version.

    • ``-g BLOCK`` flag:

      Enables the GPU version of the grayscale algorithm. This version uses CUDA and launches a kernel with a one-to-one mapping of one thread per pixel. The number of threads per block is specified by BLOCK.

      After computing the grayscale image, the GPU version must also compute a histogram of pixel intensities (0–255) and print the results to standard output.

      Each histogram bin and its corresponding count should be printed on its own line, separated by a space.

      For example:

      ./grayscale -g 16 input.png output.png

      This runs the GPU version with a block size of 16 threads per block. By default, I recommend using a block size of a warp (32 threads) or a multiple of a warp (e.g., 64, 128, 256) for better performance, but you can experiment with different block sizes to see how it affects performance.

    Assumptions

    • You do not need to perform error checking on the command-line arguments.

    • You can assume all inputs (including block sizes and filenames) are valid.

    • Your program should apply the appropriate logic based on the presence or absence of the -g flag.

    Task 1: Implement a GPU Version¶

    Inside grayscale.cu, implement a CUDA kernel named:

    image_to_grayscale_kern

    Your goal is to perform grayscale conversion on the GPU using a one-to-one mapping of threads to pixels.

    Each thread should: - Compute its global index - Map to a pixel - Convert that pixel to grayscale

    You must use the specified block size (BLOCK) when launching your kernel.

    Note

    In a future assignment, we will analyze performance and experiment with different configurations. For now, focus only on correctness.

    Verifying Correctness¶

    To test your implementation, implement a function:

    verify_gpu_results(...)

    This function should compare the GPU output against the CPU implementation provided by image_to_grayscale.

    You may define the function arguments and return type.

    The goal is to ensure your GPU implementation produces correct results.

    Floating Point Differences¶

    When comparing CPU and GPU outputs, you may observe small differences—typically within ±1 for 8-bit image values (0–255). This is expected.

    Reasons include: - Differences in floating-point hardware - Rounding behavior - Optimization strategies

    Note

    Differences of ±1 between CPU and GPU outputs are acceptable. You do not need to eliminate them.

    Histogram Computation¶

    After converting the image to grayscale on the GPU, you will compute a histogram of pixel intensities.

    What is a Histogram?¶

    A histogram represents the distribution of values in a dataset.

    For a grayscale image: - Pixel values range from 0 to 255 - The histogram counts how many pixels have each intensity

    Formally:

    int hist[256];

    where:

    • hist[i] = number of pixels with intensity i

    The total count should equal the number of pixels:

    sum(hist) == width * height

    Why is this Important?¶

    Histograms are widely used in image processing to analyze brightness and contrast:

    • Values concentrated near 0 → dark image

    • Values concentrated near 255 → bright image

    • Values spread across the range → high contrast

    Histograms summarize the distribution of values, but do not capture spatial information.

    Task 2: Compute Histogram on the GPU¶

    Inside grayscale.cu, implement a CUDA kernel named:

    compute_histogram_kern

    This kernel computes the histogram of the grayscale image.

    Requirements¶

    • Use a one-to-one mapping of threads to pixels

    • Each thread should: - Read one grayscale pixel - Increment the corresponding histogram bin

    Output Format (stdout)¶

    When running the GPU version, print the histogram to standard output.

    Each line must contain:

    <intensity> <count>

    Example:

    0 1234
1 1402
...
255 980

    There must be exactly 256 lines.

    Note

    The histogram should only be printed in the GPU version.

    Visualizing the Histogram¶

    You can visualize your histogram using gnuplot.

    First, save the output:

    ./grayscale -g 256 input.png output.png > hist.txt

    Then generate a PNG:

    gnuplot -e "
set terminal png size 800,600;
set output 'hist.png';
set title 'Grayscale Histogram';
set xlabel 'Intensity';
set ylabel 'Pixel Count';
set xrange [0:255];
set yrange [0:*];
set boxwidth 0.9;
set style fill solid;
set style data boxes;
plot 'hist.txt' using 1:2 notitle
"

    Verifying Histogram Correctness¶

    You can validate your histogram using Unix tools.

    Check total pixel count:

    ./grayscale -g 256 input.png output.png | awk '{sum += $2} END {print sum}'

    This should equal:

    width * height

    Check determinism:

    ./grayscale -g 256 input.png output.png > run1.txt
./grayscale -g 256 input.png output.png > run2.txt
diff run1.txt run2.txt

    If the outputs differ, your implementation likely contains a race condition.
