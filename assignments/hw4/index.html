    Homework #4¶

    Due: Friday, April 24th at 11:59pm

    In this assignment, you will extend your convolution-based edge detection implementation by introducing thread coarsening, and then use profiling tools to analyze its performance.

    Specifically, you will:

    1. Implement thread coarsening on your shared memory kernel

    2. Profile your kernels using NVIDIA Nsight Compute (ncu)

    3. Evaluate performance across multiple stencil sizes

    4. Analyze performance tradeoffs using real profiling data

    You will perform experiments using the following stencil sizes:

    • 3x3

    • 5x5

    • 7x7

    Command-Line Interface¶

    Your program now supports an optional coarsening factor:

    ./edge_detect global|shared stencil.txt input.png output.png [coarse_factor]

    • coarse_factor is optional

    • Default value: 1 (no coarsening use your previous shared-memory kernel from hw3)

    • Coarsening applies only to the shared memory kernel

    • You do not need to error check your arguments. You can assume we will always give you correct arguments. This means we will never give you a coarse_factor argument when running the global version. We will also always provide a positive integer greater than or equal to 1 for the coarse_factor.

    Example usage:

    ./edge_detect global stencil3.txt in.png out.png
./edge_detect shared stencil3.txt in.png out.png 1
./edge_detect shared stencil3.txt in.png out.png 2
./edge_detect shared stencil3.txt in.png out.png 4

    Task 1 — Thread Coarsening¶

    Concept¶

    In your current implementation:

    Each thread computes one output pixel

    You will modify this so that:

    Each thread computes multiple output pixels

    This technique is called thread coarsening. Please watch the videos for week 4 where I explain thread coarsening.

    Requirements¶

    • Implement a new kenerl that performs thread coarsening in your using shared memory.

    • Each thread computes coarse_factor outputs

    • Your implementation must work for all stencil sizes (3x3, 5x5, 7x7)

    Example Pattern¶

    for (int c = 0; c < coarse_factor; c++) {
    int out_col = base_col + c * blockDim.x;
    if (out_col < width) {
        // compute convolution
    }
}

    Important Considerations¶

    • Update grid dimensions appropriately:

      gridDim.x = ceil(width / (blockDim.x * coarse_factor))

    • Ensure correct indexing

    • Handle boundary conditions carefully

    • Reuse shared memory efficiently

    • Avoid redundant global memory accesses

    Task 2 — Profiling Baseline Kernels¶

    You should already have:

    • A global memory convolution kernel

    • A shared memory (tiled) convolution kernel

    Experimental Setup¶

    To ensure consistent and meaningful performance results, you must use the provided input image for all profiling experiments.

    Requirements:

    • Use the same input image for all runs

    • Use one image that is sufficiently large (e.g., 2048×2048 or larger). You may need to use your own image and please make sure to add it to the repo.

    • Do not vary image size when comparing stencil sizes or coarsening factors

    Rationale:

    • Small images may not fully utilize the GPU

    • Large images ensure that performance differences reflect kernel behavior rather than overhead

    Running the Profiler¶

    Use:

    ncu --set full ./edge_detect [args]

    Task 4 — Analysis & Discussion¶

    Answer the following:

    1. Optimal Coarsening

      • What is the best coarse factor for each stencil size?

    2. Performance Trends

      • How does performance change as coarse factor increases?

      • Why does performance eventually degrade?

    3. Occupancy Tradeoff

      • How does occupancy change with increasing coarse factor?

      • Does higher occupancy always lead to better performance?

    4. Memory Behavior

      • Does coarsening significantly change memory throughput?

      • Why or why not?

    5. Scaling Insight

      • Does the optimal coarse factor depend on stencil size?

      • Why might larger stencils benefit more from coarsening?

    6. Final Insight

      • What combination of techniques gave the best performance?

      • Explain the tradeoffs involved
